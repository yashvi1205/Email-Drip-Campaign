Merged Sales Automation Pipeline

Complete system flow · Excel Queue → AI Research → Email Drafting → Gmail Send → Real-Time Tracking

▶  System Starts
⚡ Stage A  ·  AI Qualification & Scoring
1 Read Leads from Excel Queue
Opens Sales_Automation_Pipeline.xlsx and scans every row.
Picks up rows where Include in Qualify Batch is set to Yes and no score has been assigned yet.
Collects: company website URL, name, and contact email for each lead.
2 Scrape the Prospect's Website
Visits the company's website and fetches multiple pages in parallel:
  • Homepage – company overview & messaging
  • /about – team size, mission, leadership names
  • /pricing – pricing model (confirms budget capacity)
  • /careers – open roles (signals growth & hiring activity)
  • /contact – contact details & regions
Also detects the tech stack they use (Stripe, HubSpot, Shopify, React…)
3 Codex AI Analyses the Scraped Data
All scraped page content is sent to Google Codex AI.
Codex identifies signals across two qualification frameworks:
  • BANT – Budget, Authority, Need, Timeline
  • MEDDIC – Metrics, Economic Buyer, Decision Criteria, Decision Process, Identify Pain, Champion
Generates a full LEAD-QUALIFICATION.md report with findings.
4 Score & Grade the Lead (0–100)
Runs lead_scorer.py on the qualification output.
Computes a numeric fit score from 0 to 100 and assigns a grade:
  • A (75–100) – Strong fit, high priority
  • B (50–74) – Good fit, worth pursuing
  • C (25–49) – Weak fit, low priority
  • D (0–24) – Poor fit, skip
Writes back to Excel: Score · Grade · Report file link · Status → Pending Qualify Review
🛑  Human Review Gate 1 — Qualification Approval
You open Excel · Read the qualification report · Check the score and grade.
Set the Qualify Approval column to Approved or Rejected.
✗  Rejected
🗑️ Archive Lead
Status = Disqualified
✓  Approved
⬇ Move to Stage B
✉️ Stage B  ·  Outreach Email Drafting
5 Load the Qualification Report
Reads the LEAD-QUALIFICATION.md file generated in Stage A.
Extracts personalization anchors:
  • Company pains & challenges
  • Decision-maker names and titles
  • Technology gaps or integrations they are missing
  • Budget & timeline signals
6 Codex Writes a 5-Email Outreach Sequence
Passes the personalization anchors to Codex AI with a structured prompt.
AI drafts a complete drip sequence:
  • Email 1 (Day 1) – Hook: a direct, pain-focused opening line
  • Email 2 (Day 3) – Value: share a relevant resource or insight
  • Email 3 (Day 7) – Proof: a real case study or result
  • Email 4 (Day 14) – Angle: a completely different perspective
  • Email 5 (Day 21) – Breakup: polite final check-in
Saves as OUTREACH-SEQUENCE.md · Updates Excel: Draft link · Status → Pending Email Review
🛑  Human Review Gate 2 — Email Copy Approval
You read all 5 emails · Edit the Markdown file if anything needs changing.
Set Email Approval column to Approved or Rejected.
✗  Rejected
✏️ Edit Email Draft
Revise & re-approve
✓  Approved
⬇ Move to Stage C
📧 Stage C  ·  Tracked Email Send
7 Generate Tracking ID & Inject Trackers
Creates a unique Tracking ID (e.g. trk_1719647382_ab3c) for this send.
Injects an invisible 1×1 pixel image into the email:
https://render-server.com/api/tracking/track/trk_id
Wraps every clickable link through a redirect URL:
https://render-server.com/api/tracking/click/trk_id?url=original_link
8 Send Email 1 via Gmail API
Authenticates using OAuth 2.0 and calls the Gmail API to deliver the email.
Writes to Excel: Sent Date timestamp · Status → Sent
Subsequent follow-up emails (2–5) are scheduled based on the day offsets.
📡 Stage D  ·  Real-Time Tracking & Excel Sync
9 Recipient Opens the Email or Clicks a Link
When the recipient's email client loads the email, it automatically loads the pixel image.
When they click any link, they are briefly redirected through the tracking URL.
10 Request Hits the Public Render Server
The pixel load or link click sends a live HTTP request to the Render.com API server (your production backend).
Render receives the tracking ID and event type: open or click.
11 Render Forwards Event via untun Tunnel
Render sends the event data to your local machine via the untun HTTPS tunnel.
This bridges the public Render server → your private localhost server without exposing any ports publicly.
12 Bot Filter — Is This a Real Person?
The local backend checks the User-Agent header of the incoming request.
  • 🚫 Blocks known scanner bots, security crawlers, and link checkers
  • ✅ Allows googleimageproxy — this is how Gmail registers email opens
  • ✅ Allows real browser User-Agents (Chrome, Firefox, Safari)
13 Log Event in Local PostgreSQL Database
Records a new row in the tracking_events table in your local Postgres database.
Stores: event type · tracking ID · timestamp · IP address · User-Agent.
Increments open_count / click_count on the email_sequences record.
14 Update Excel Open & Click Counters
Matches the tracked email back to the correct row in Sales_Automation_Pipeline.xlsx.
Matching is done by normalised email address, LinkedIn URL, or company name.
Increments the open_count and click_count cells directly in that row.
Updates Last Updated timestamp column.
✅  Cycle Complete — Next Email in Sequence Scheduled
